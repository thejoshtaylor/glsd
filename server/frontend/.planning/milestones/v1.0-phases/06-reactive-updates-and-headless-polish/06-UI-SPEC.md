## Interaction States

This phase modifies two interaction flows. No new UI surfaces are created; the contract governs state transitions in existing components.

### Reactive Invalidation (Worktrees + Visualizer)

| Trigger | Visual Response | Timing |
|---------|----------------|--------|
| `.gsd/` file changes while Worktrees tab is open | Worktree list enters TanStack Query refetch state — existing loading skeleton shows if stale-while-revalidate threshold exceeded | Within 2 seconds of file change |
| `.gsd/` file changes while Visualizer tab is open | Visualizer data enters TanStack Query refetch state — no loading flicker if data is fresh (stale-while-revalidate) | Within 2 seconds of file change |

No new loading states are needed. Both tabs already render skeletons during query fetch. The fix is invisible to the user except for faster data freshness.

### Headless Log Persistence

| State | Before Phase 6 | After Phase 6 |
|-------|----------------|---------------|
| Navigate away from Headless tab during active session | All `logs[]` rows destroyed on tab unmount | `logs[]` survive — hook state held at `ProjectPage` scope |
| Navigate back to Headless tab | Empty log area; new rows begin streaming from resume point | All historical rows present; new rows append normally |
| Session is idle when navigating back | Empty state: "No session started" | Unchanged — same empty state |

Interaction contract for log buffer display:
- Log area shows all rows accumulated since the last `clearLogs()` call (triggered only by Start Session)
- Auto-scroll to bottom on new row append — maintained via `scrollRef.current.scrollTop`
- Stop Session does NOT clear logs — user can read the last session's output after stopping
- Start Session clears logs (calls `clearLogs()`) before beginning new session

### Empty State

| Condition | Heading | Body |
|-----------|---------|------|
| No session started (status is idle, no logs) | "No session started" | "Click Start Session to run gsd headless for this project." |
| Session running, logs not yet received | (no heading — Skeleton rows shown instead) | (no body) |

Source: `src/components/project/gsd2-headless-tab.tsx` lines 159-178.

## Phase 6 Scope Boundary

This phase introduces zero new visual surfaces. The UI contract governs:

1. **Reactive behavior** — Worktrees and Visualizer query invalidation on `gsd2:file-changed` (no visible UI change; just faster data refresh)
2. **Log state persistence** — existing `Gsd2HeadlessTab` log buffer survives tab navigation via prop-lift refactor
3. **Documentation** — ROADMAP.md progress table row, two SUMMARY frontmatter fields (no UI at all)

Any visual changes are incidental to the refactor (e.g., logs that were previously lost now appear). The appearance of the Headless tab is unchanged — only its state lifecycle changes.
