## Color

Design system uses HSL CSS variables with light and dark mode. Application primary mode is dark (Tauri desktop app; dark class is active by default).

| Role | HSL Variable | Dark Mode Value | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `222 20% 4.5%` | App background, main surfaces |
| Secondary (30%) | `--card` / `--secondary` | `222 18% 7%` / `222 16% 14%` | Cards, sidebar, nav, panels |
| Accent (10%) | `--primary` / `--brand-blue` | `217 91% 65%` | See reserved list below |
| Destructive | `--destructive` | `0 62.8% 30.6%` | Destructive actions only |

Accent (`--primary` / `--brand-blue`) is reserved for:
- Active navigation item highlight
- Focus ring (`--ring`)
- Primary button backgrounds
- Brand-identity gradient text (`text-gradient` utility)
- GSD version badge border color (Phase 2, not this phase)

Status semantic colors (from `tailwind.config.js:62-70`):
- `status-success` — completed states
- `status-warning` — warning states
- `status-error` — error/failed states (also maps to destructive text)
- `status-info` — in-progress / running states (matches `--brand-blue`)
- `status-pending` — pending/idle states
- `status-blocked` — blocked states (orange, distinct from warning)
- `status-paused` — paused states (yellow, distinct from blocked)

Source: `src/styles/globals.css:50-85` (dark mode token values).

## Error Display Contract (Version Mismatch Guard Rail)

This is the only frontend-relevant output from Phase 1. When a GSD-1 Rust command is called against a GSD-2 project, it returns `Err("This project uses GSD-2. Use gsd2_* commands instead.")`.

The existing frontend error handling pattern (via `ErrorBoundary` and Tauri `invoke()` catch blocks) ALREADY handles `Err(String)` from Tauri commands. No new error UI component is needed.

**Contract:** The error string must pass through the existing error display path unmodified. No new toast, modal, or alert component is introduced in this phase. Phase 2 will own any UI surface for surfacing version state to users.
