This avoids dynamic hook count issues (hooks can't be in loops).

**BUT — to avoid fetching ALL slices eagerly (could be 25+ queries), implement a progressive approach:**
1. First render: show only slices from the active milestone (derived from `useGsd2DerivedState`)
2. Use a "Show all milestones" toggle button if there are other milestones with non-done slices

**Simplified approach (recommended — avoids complexity):**
- Since the Tasks tab per CONTEXT.md shows "active + pending tasks only", and the user's primary interest is the current work:
- Fetch slices only for the active milestone initially using `useGsd2Milestone(projectId, activeMilestoneId, !!activeMilestoneId)` which returns the full milestone with slice metadata
