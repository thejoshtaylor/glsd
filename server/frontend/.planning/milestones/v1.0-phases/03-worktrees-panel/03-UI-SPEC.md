## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | gap-1, p-1 | Icon gaps, inline badge padding |
| sm | 8px | gap-2, p-2 | Row inner padding, icon-to-text gaps |
| md | 16px | gap-4, p-4 | Card content padding (CardContent default) |
| lg | 24px | gap-6, p-6 | Section vertical spacing within tab |
| xl | 32px | gap-8 | Major section breaks (not used in this phase) |
| 2xl | 48px | py-12 | Empty state vertical centering |
| 3xl | 64px | — | Page-level only, not used in this phase |

Exceptions:
- CardHeader uses `pb-3` (12px) — matches existing health tab pattern (source: gsd2-health-tab.tsx line 23)
- Row accordion chevron button: `h-4 w-4` icon at 16px — consistent with health tab icon sizing
- Compact file list rows: `py-2` (8px) vertical — declared value is grid-compliant (multiple of 4). The app's `--spacing-scale: 0.8` reduces the rendered output to 6.4px, achieving target compact density while keeping the declared token on-grid.
- Diff file row items: `py-1` (4px) vertical — declared value is grid-compliant (multiple of 4). Rendered at 3.2px under the 0.8 scale, matching the tight list density of git-status-widget.
- Spacing scale multiplier: app has `--spacing-scale: 0.8` (compact density default) — all spacing is rendered at 80% of Tailwind defaults

## Layout Contract

### Tab Placement

Position: second tab in GSD-2 sub-tab group.
Order: Health → **Worktrees** → Milestones → Slices → Tasks
Visibility: always rendered for GSD-2 projects regardless of worktree count.
Source: CONTEXT.md — Tab Placement decision (locked).

### Worktree List Row (collapsed state)

```
[ChevronRight] [GitBranch icon] {name}    [branch badge]   [+N added · M modified · K removed]   [Remove button]
               font-mono text-sm          variant=outline   text-xs, color-coded per status       variant=destructive size=sm
```

- Row height: `py-3 px-4` — 12px vertical, 16px horizontal padding
- Name: `text-sm font-mono font-medium text-foreground`
- Branch badge: `<Badge variant="outline" className="font-mono text-xs">` — branch name text
- Change counts: `text-xs` inline, `text-status-success` for +N added, `text-status-warning` for M modified, `text-status-error` for K removed, separated by ` · ` (middot) in `text-muted-foreground`
- Path: `text-xs font-mono text-muted-foreground` — truncated with `truncate`, full path in `title` attribute
- Remove button: always visible, right-aligned, `variant="destructive" size="sm"` with `<Trash2 className="h-3.5 w-3.5" />`

### Worktree Row (expanded state)

Inline accordion — same row, chevron rotates from Right to Down (`transition-transform duration-200`).
Expanded content renders below the row header inside the same card border, separated by a `border-t`.

Diff section layout (three groups, each collapsible):
```
[FilePlus icon] Added ({count})          — text-status-success, text-xs font-medium
  filename.ts                            — text-xs font-mono text-status-success/80, py-1 pl-6
  ...

[FilePenLine icon] Modified ({count})    — text-status-warning, text-xs font-medium
  filename.ts                            — text-xs font-mono text-status-warning/80, py-1 pl-6

[FileMinus icon] Removed ({count})       — text-status-error, text-xs font-medium
  filename.ts                            — text-xs font-mono text-status-error/80, py-1 pl-6
```

Note: `py-1` (4px declared) renders at 3.2px under the app's `--spacing-scale: 0.8`, producing the tight list density that matches git-status-widget. The declared token is grid-compliant.

Groups with zero files are omitted entirely.
File list wrapped in ScrollArea with `max-h-48` when count > 8.
Lazy loading: diff data fetched only when row is first expanded (React Query enabled: false until open).

### Empty State

Centered vertically within the card content area with `py-12 text-center`.
```
No active worktrees
GSD-2 creates these automatically when running parallel slices.
```
- Heading: `text-sm font-medium text-muted-foreground`
- Body: `text-xs text-muted-foreground mt-1`
- No illustration, no action buttons.
Source: CONTEXT.md — Empty State decision (locked).

### Loading State

Three skeleton rows inside CardContent:
```
<Skeleton className="h-12 w-full" />   x3, space-y-3
```
Matches health tab skeleton pattern (source: gsd2-health-tab.tsx lines 28–30).

## Interaction Contract

### Accordion (row expand/collapse)

- Click anywhere on the row header area (excluding Remove button) toggles the accordion.
- Only one row expanded at a time is NOT required — multiple rows may be open simultaneously.
- Chevron rotates: `ChevronRight` → `ChevronDown` on open, `transition-transform duration-200`.
- Expand toggle button: `variant="ghost" size="icon"` with `aria-label="Expand {worktree-name} diff"` (substitute actual worktree name at render time, e.g. `aria-label={\`Expand ${worktree.name} diff\`}`). Update `aria-label` to `"Collapse {worktree-name} diff"` when the row is open.
- Diff data fetches on first expand; subsequent opens use React Query cache (stale time: 30s).

### Remove Action

1. User clicks Remove button → `AlertDialog` opens immediately (no delay).
2. AlertDialog shows worktree name and branch name. Cancel / Remove buttons.
3. User clicks Remove in dialog → optimistic update fires:
   - Row removed from local list immediately (`queryClient.setQueryData` removes entry).
   - `gsd2_remove_worktree` mutation fires in background.
4. On success: query invalidated, list re-fetches to confirm removal.
5. On error: row reappears (query invalidated → re-fetch restores original state), error toast displayed via `sonner` toast.error.
6. Remove button disabled while mutation is pending (shows Loader2 spinner).
Source: CONTEXT.md — Remove Confirmation + Specific Ideas (locked).

### Tab Load Behavior

- `useGsd2Worktrees(projectId)` fires on tab mount.
- Change counts included in `gsd2_list_worktrees` response (eagerly loaded) — no separate diff call on load.
- `useGsd2WorktreeDiff(projectId, name)` fires only when a row is expanded (enabled: boolean state).
