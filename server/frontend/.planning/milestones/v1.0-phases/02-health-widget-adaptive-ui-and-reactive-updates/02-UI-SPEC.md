## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), tight inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`), badge padding |
| md | 16px | Default card content spacing (`space-y-4`, `p-4`) |
| lg | 24px | Section padding, card header padding (`pb-3` = 12px threshold) |
| xl | 32px | Layout gaps between widget cards |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in widget views) |

**Compact density is the project default** (`--spacing-scale: 0.8`). The health widget follows the same compact density as the git-status-widget reference (`space-y-3`, `CardHeader pb-3`, `CardContent space-y-3`).

Exceptions:
- Blocker row inner padding: `p-2` (8px) — tighter than standard `md` to keep the alert contained
- Progress counters grid: `gap-2` (8px) between the 3-column cells — source: RESEARCH.md component skeleton
- Touch targets: not applicable (desktop Tauri app, no minimum touch target requirement)

| Role | Token | Dark value (hsl) | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `hsl(var(--background))` | 222 20% 4.5% | Page background, tab content area |
| Secondary (30%) | `hsl(var(--card))` | 222 18% 7% | Health widget Card container, progress counter cells |
| Accent (10%) | `hsl(var(--brand-blue))` | 217 91% 65% | Budget progress bar fill (brand variant), active state ring, primary action highlights |
| Destructive | `hsl(var(--destructive))` / `hsl(var(--status-error))` | 0 62.8% 30.6% / 0 88% 68% | Destructive actions only (none in this phase) |
| Status — Error | `hsl(var(--status-error))` | 0 88% 68% | Blocker row border, icon, and text: `border-status-error/20 bg-status-error/10 text-status-error` |
| Status — Warning | `hsl(var(--status-warning))` | 38 95% 58% | Budget bar when `budgetPct > 80` (Progress `variant="warning"`) |
| Brand Cyan | `hsl(var(--brand-cyan))` | 189 94% 50% | GSD-2 version badge (`variant="subtle-cyan"`) |
| Secondary muted | `hsl(var(--secondary))` | 222 16% 14% | GSD-1 version badge (`variant="secondary"`) |
| Muted foreground | `hsl(var(--muted-foreground))` | 220 8% 65% | Counter sub-labels ("Milestones", "Slices", "Tasks"), empty state text |

**Accent reserved for:** budget progress bar fill, `brand` Progress indicator gradient, focus rings. Never used for generic text or borders.

**Blocker row color contract:** `border border-status-error/20 bg-status-error/10 text-status-error` — this is the exact Tailwind class set. The row is conditionally rendered only when `health.blocker` is non-null.

**Version badge color contract:**
- GSD-2 badge: `variant="subtle-cyan" size="sm"` → `border-brand-cyan/20 bg-brand-cyan/10 text-brand-cyan`
- GSD-1 badge: `variant="secondary" size="sm"` → `bg-secondary text-secondary-foreground`
- Non-GSD projects: no badge rendered (clean omission)

Source: CONTEXT.md Decisions + badge.tsx variant inspection.

## Layout Contract

### Health Widget (`Gsd2HealthTab`)

Structure: `Card > CardHeader + CardContent`

```
Card
└─ CardHeader (pb-3)
   └─ CardTitle (text-sm font-semibold, flex items-center gap-2)
      ├─ Activity icon (h-4 w-4)
      └─ "GSD Health"
└─ CardContent (space-y-3)
   ├─ Budget row (div)
   │  ├─ flex justify-between text-xs mb-1
   │  │  ├─ "Budget" label
   │  │  └─ "{formatCost(spent)}" [/ "{formatCost(ceiling)}"]
   │  └─ Progress (variant="brand"|"warning", size="sm") — omitted if no ceiling
   ├─ Blocker row (conditional — hidden when no blocker)
   │  └─ flex items-center gap-2 p-2 rounded border border-status-error/20 bg-status-error/10
   │     ├─ AlertCircle (h-4 w-4 text-status-error flex-shrink-0)
   │     └─ span (text-sm text-status-error) {health.blocker}
   ├─ Active unit row (div — shown when active_milestone_id is non-null)
   │  └─ text-xs space-y-1
   │     ├─ Milestone: "{active_milestone_id} — {active_milestone_title}"
   │     ├─ Slice: "{active_slice_id} — {active_slice_title}" | "None"
   │     └─ Phase: "{phase}"
   └─ Progress counters (grid grid-cols-3 gap-2 text-xs text-center)
      ├─ Milestones: "{done}/{total}" font-semibold + "Milestones" text-muted-foreground
      ├─ Slices: "{done}/{total}" font-semibold + "Slices" text-muted-foreground
      └─ Tasks: "{done}/{total}" font-semibold + "Tasks" text-muted-foreground
```

**Active unit display:** Show both ID and title (e.g., "M001 — End-to-End Testing"). Source: CONTEXT.md "Claude's Discretion" — choosing ID + title to be maximally informative. Use `text-xs` for the entire active unit section to keep it compact.

**ETA display:** Use `formatRelativeTime()` from `src/lib/utils.ts` for relative time (e.g., "in 2 hours"). Only render the ETA row when `health.eta` is non-null. Source: CONTEXT.md "Claude's Discretion" — relative time chosen for immediacy.

**Env error/warning counts:** Render only when non-zero. Use `Badge variant="error" size="sm"` for error count and `Badge variant="warning" size="sm"` for warning count. If both are 0 (current default), the env row is omitted entirely. Source: RESEARCH.md open question 3 — env counts default to 0.

### Version Badge Placement

**`src/components/projects/project-card.tsx` Row 4 (tech stack row):**
Insert badge after the existing tech-stack span elements. The badge goes last in the row, separated by the same implicit `gap` already used by the row. No layout change to surrounding elements.

**`src/components/dashboard/project-card.tsx`:**
Insert badge in the GSD live-stats row or near the project type chip in the card header. Exact insertion point is the GSD stats area — match the project-card.tsx Row 4 pattern as closely as the dashboard card structure allows.

### GSD Tab Set Switching

The top-level tab trigger guard changes from `hasPlanning` to `showGsdTab = isGsd2 || isGsd1`.

GSD-2 sub-tab order (locked from CONTEXT.md): Health, Milestones, Slices, Tasks.

Default active tab for GSD-2 projects: `gsd2-health` (Health tab).

GSD-1 sub-tab set: unchanged — no modifications to any existing GSD-1 tab component or its order.

## Interaction Contract

### Polling and Reactivity (HLTH-04)

- Health data refetches every 10 seconds (`refetchInterval: 10_000`).
- `gsd2:file-changed` events from the Tauri watcher immediately invalidate `queryKeys.gsd2Health(projectId)` — no debounce.
- No manual refresh button on the health widget (data is reactive; unlike git widget which has an explicit refresh control).
- `staleTime: 5_000` — data considered fresh for 5 seconds between background refetches.

### Loading State

Show `Skeleton` rows while `isLoading` is true (matches git-status-widget loading pattern):

```tsx
<Card>
  <CardHeader className="pb-3">
    <CardTitle className="text-sm font-semibold flex items-center gap-2">
      <Activity className="h-4 w-4" /> GSD Health
    </CardTitle>
  </CardHeader>
  <CardContent className="space-y-3">
    <Skeleton className="h-4 w-2/3" />
    <Skeleton className="h-4 w-1/2" />
    <Skeleton className="h-4 w-3/4" />
  </CardContent>
</Card>
```

### Error State

If `useGsd2Health` returns an error (Rust command failed), show inline error in CardContent:

```tsx
<CardContent className="py-4 text-center text-sm text-status-error">
  Failed to load health data — check that the project path is accessible.
</CardContent>
```

No toast notification for polling errors — silent retry on next interval.

### Tab Visibility Logic

```
isGsd2 = project.gsd_version === 'gsd2'
isGsd1 = hasPlanning && !isGsd2
showGsdTab = isGsd2 || isGsd1
```

The GSD TabsTrigger renders only when `showGsdTab` is true. Inside TabsContent, render the GSD-2 TabGroup when `isGsd2`, otherwise render the existing GSD-1 TabGroup when `isGsd1`. These branches are mutually exclusive.

## Accessibility Contract

- All icon-only elements must have a `title` or be paired with visible text. The `Activity` and `AlertCircle` icons in the health widget are decorative (paired with text) — no `aria-label` required.
- Badge elements are `<div>` (existing badge.tsx behavior) — ensure parent context provides semantic meaning.
- Progress bars from Radix UI (`@radix-ui/react-progress`) include `role="progressbar"` with `aria-valuenow` and `aria-valuemax` automatically. Always pass `value` prop as a number 0–100.
- The blocker row does not use `role="alert"` — it is a static display element, not a live region. No auto-announced content.
- Keyboard navigation: tab set uses the existing `TabGroup` component which handles keyboard arrow navigation via Radix Tabs — no additional keyboard contract needed.
- Reduced motion: existing `globals.css` handles `prefers-reduced-motion` — Progress transition (`transition-all duration-500`) will be disabled automatically for users with this preference set.
