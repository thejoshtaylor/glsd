New module with HeadlessSessionRegistry. Include file header. Contents:

```rust
use std::collections::HashMap;
use std::sync::Arc;
use tokio::sync::Mutex;

pub struct HeadlessSessionRegistry {
    /// session_id -> project_id mapping
    pub sessions: HashMap<String, i64>,
}

impl HeadlessSessionRegistry {
    pub fn new() -> Self {
        Self { sessions: HashMap::new() }
    }

    pub fn register(&mut self, session_id: String, project_id: i64) {
        self.sessions.insert(session_id, project_id);
    }

    pub fn unregister(&mut self, session_id: &str) {
        self.sessions.remove(session_id);
    }

    pub fn get_project_id(&self, session_id: &str) -> Option<i64> {
        self.sessions.get(session_id).copied()
    }

    pub fn session_for_project(&self, project_id: i64) -> Option<String> {
        self.sessions.iter()
            .find(|(_, &pid)| pid == project_id)
            .map(|(sid, _)| sid.clone())
    }

    pub fn all_session_ids(&self) -> Vec<String> {
        self.sessions.keys().cloned().collect()
    }

    pub fn active_count(&self) -> usize {
        self.sessions.len()
    }
}

pub type HeadlessRegistryState = Arc<Mutex<HeadlessSessionRegistry>>;
```

Add `gsd2_headless_start` command — parameters: `app: tauri::AppHandle`, `project_id: String`, `pool: State<'_, DbState>`, `terminal_manager: State<'_, crate::pty::TerminalManagerState>`, `registry: State<'_, HeadlessRegistryState>`. Steps:
  1. Check registry for existing session for this project — if one exists, return error "A headless session is already running for this project"
  2. Get project path from DB (pool.read()), drop DB lock
  3. Generate session_id via `uuid::Uuid::new_v4().to_string()`
  4. Lock terminal_manager (separate scope), call `create_session(&app, session_id.clone(), &project_path, Some("gsd headless"), 80, 24)`
  5. Lock registry (separate scope), call `register(session_id.clone(), project_id_i64)`
  6. Return session_id

Add `gsd2_headless_stop` command — parameters: `app: tauri::AppHandle`, `session_id: String`, `terminal_manager: State<'_, crate::pty::TerminalManagerState>`, `registry: State<'_, HeadlessRegistryState>`. CRITICAL: `app: tauri::AppHandle` MUST be the first parameter — it is required by `TerminalManager::close(&app, &session_id)` for force-kill. Steps:
  1. Lock terminal_manager, write `&[0x03]` (ETX/Ctrl-C), release lock
  2. Poll loop: 25 iterations, each `tokio::time::sleep(Duration::from_millis(200))`, check `manager.is_active(&session_id)`, break if not active
  3. If still active after loop: lock terminal_manager, call `close(&app, &session_id)`, release lock
  4. Lock registry, call `unregister(&session_id)`, release lock
  5. Return `Ok(())`

CRITICAL: Never hold both `terminal_manager` and `registry` locks simultaneously. Use separate `{}` scopes for each lock acquisition.

Add `can_safely_close` command — parameters: `terminal_manager: State<'_, crate::pty::TerminalManagerState>`, `registry: State<'_, HeadlessRegistryState>`. Returns `ActiveProcessInfo { can_close: bool, active_terminals: usize }`:

Add `gsd2_get_visualizer_data` command — parameters: `project_id: String`, `pool: State<'_, DbState>`. Steps:
  1. Get project path from DB (pool.read()), drop DB lock
  2. Build the milestone directory path: `{project_path}/.gsd/milestones`
  3. Call `walk_milestones_with_tasks(&milestones_dir)` to get all milestones with slices and tasks
  4. Call existing `get_health_from_dir(project_path)` to get active milestone/slice/task IDs for status tagging
  5. Build the `tree: Vec<VisualizerNode>` — for each milestone, create a VisualizerNode with status derived from: done if `milestone.done == true`, active if `milestone.id == active_milestone_id`, else pending. For each slice in the milestone, same logic with `active_slice_id`. For each task in the slice, same logic with `active_task_id`. Children of milestone = slice nodes; children of slice = task nodes.
  6. Parse `metrics.json` at `{project_path}/.gsd/metrics.json` — read the file, parse as JSON. Access `units` array. For each unit entry:
     - Get `milestone_id`: `unit.get("milestone_id").and_then(|v| v.as_str()).unwrap_or("unattributed")`
     - Get `model`: `unit.get("model").and_then(|v| v.as_str()).unwrap_or("unknown")`
     - Get `cost`: `unit.get("cost").and_then(|v| v.as_f64()).unwrap_or(0.0)`
     - Get `completed_at`: `unit.get("completed_at").and_then(|v| v.as_str()).map(String::from)`
     - Get `id`: `unit.get("id").and_then(|v| v.as_str()).unwrap_or("").to_string()`
     - Get `title`: `unit.get("title").and_then(|v| v.as_str()).unwrap_or("").to_string()`
     - Get `entry_type`: `unit.get("type").and_then(|v| v.as_str()).unwrap_or("task").to_string()`
  7. Aggregate `cost_by_milestone`: HashMap<String, f64> grouping costs by milestone_id, convert to Vec<CostByKey> sorted by key
  8. Aggregate `cost_by_model`: HashMap<String, f64> grouping costs by model, convert to Vec<CostByKey> sorted by cost descending
  9. Build `timeline`: Vec<TimelineEntry> from units that have `completed_at` set, sorted by completed_at descending (most recent first)
  10. If metrics.json doesn't exist or fails to parse, return empty vectors for cost_by_milestone, cost_by_model, and timeline (NOT an error)
  11. Return `VisualizerData { tree, cost_by_milestone, cost_by_model, timeline }`
