## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding, log row vertical padding (`py-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `space-y-2`), card header bottom padding (`pb-2`) |
| md | 16px | Default element spacing (`gap-4`, `space-y-4`), card content padding (`p-4`) |
| lg | 24px | Section padding (`p-6`), card header padding (`pb-3` = 12px exception — see below) |
| xl | 32px | Layout gaps between major sections (tree, cost charts, timeline) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Card header `pb-3` (12px) is used in existing GSD-2 tabs (health, worktrees) — maintain for visual consistency.
- Monospace timestamp column: `w-20` (80px fixed width); cost delta column: `w-20` (80px fixed width).

Source: gsd2-health-tab.tsx patterns; CONTEXT.md specifics.

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Inter var | Card descriptions, status labels, list item text |
| Label | 12px (`text-xs`) | 400 (regular) | 1.4 | Inter var | Log rows, cost bar labels, timeline timestamps, muted metadata |
| Heading | 14px (`text-sm`) | 600 (semibold) | 1.2 | Inter var | Card titles (`font-semibold`), section headers |
| Mono | 12px (`text-xs font-mono`) | 400 (regular) | 1.4 | JetBrains Mono Variable | Headless log rows (`[HH:MM:SS]  {state}  +$X.XXX`), session IDs |

Notes:
- Only 2 weights used project-wide: 400 (regular) and 600 (semibold). Do not introduce 500 or 700.
- Mono is reserved for: log row timestamps, log row state strings, log row cost deltas, session ID display.
- `text-xs` (12px) is the floor — no smaller text.

Source: gsd2-health-tab.tsx, tailwind.config.js fontFamily, RESEARCH.md Pattern 6 (LogRow component).

## Color

| Role | Token | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `background` | `hsl(var(--background))` | Page background, tab content areas |
| Secondary (30%) | `card` / `muted` | `hsl(var(--card))` / `hsl(var(--muted))` | Card surfaces, snapshot card, log row background, bar chart track |
| Accent (10%) | `primary` | `hsl(var(--primary))` = HSL 217 91% 60% (blue) | Bar chart fill, active progress indicator, Start button |
| Destructive | `destructive` | `hsl(var(--destructive))` | Stop button (session termination) |

Dark mode values (project is primarily used in dark mode):
- `--background`: HSL 222 20% 4.5% (near-black)
- `--card`: HSL 222 18% 7% (dark card surface)
- `--muted`: HSL 222 16% 14% (muted surface, log row track, bar chart bg)
- `--primary`: HSL 217 91% 60% (brand blue — same in both modes)

Accent reserved for ONLY these specific elements:
1. Start Session button background
2. Bar chart fill (cost bars in Visualizer)
3. Active session status dot color
4. Active node `▶` icon color (use `text-yellow-500` per CONTEXT.md — NOT primary blue)

Status color assignments for headless session states:
- Idle: `text-muted-foreground` dot (gray — `bg-muted-foreground`)
- Running: `text-status-success` dot with `animate-pulse` (green pulsing — `bg-status-success`)
- Complete: `text-status-success` dot solid (green — `bg-status-success`)
- Failed: `text-status-error` dot (red — `bg-status-error`)

Status color assignments for Visualizer tree nodes (from CONTEXT.md decisions):
- Done: `text-status-success` — checkmark `✔`
- Active: `text-yellow-500 animate-pulse` — play triangle `▶`
- Pending: `text-muted-foreground` — circle `○`

Log row cost delta: `text-status-success` (green `+$X.XXX`). Source: RESEARCH.md Pattern 8 (LogRow).

Source: globals.css, tailwind.config.js, CONTEXT.md decisions, gsd2-health-tab.tsx, RESEARCH.md Pattern 8.

## Layout Contracts

### Headless Tab Layout

```
+--------------------------------------------------+
| [● Running]  Start Session  Stop Session         |  <- Status bar (flex row, items-center, gap-2)
+--------------------------------------------------+
| SNAPSHOT CARD (pinned, always visible)           |
|  State: {state string}                           |
|  Next:  {next string or "—"}                     |
|  Cost:  {total running cost}                     |
+--------------------------------------------------+
| SESSION LOG (ScrollArea, flex-1, auto-scroll)    |
|  [HH:MM:SS]  {state}          +$X.XXX           |  <- LogRow (font-mono text-xs)
|  [HH:MM:SS]  {state}          +$X.XXX           |
|  ...                                             |
+--------------------------------------------------+
```

- Status bar: `flex items-center gap-2 mb-4`
- Status dot: `h-2 w-2 rounded-full` with state-specific color + optional `animate-pulse`
- Status label: `text-sm font-semibold` (e.g. "Running", "Idle")
- Cost-so-far shown inline: `text-xs text-muted-foreground ml-auto` when running; last-run time when idle/complete
- Snapshot card: `Card` variant default; three rows label + value; `text-sm`
- Log area: `ScrollArea` with `max-h-96` (384px) or `flex-1` in full-height layouts; `space-y-0` (rows are dense)
- Log row: `flex items-center text-xs font-mono py-1 hover:bg-muted/30`
  - Timestamp: `w-20 text-muted-foreground shrink-0`
  - State: `flex-1 truncate px-2`
  - Cost delta: `w-20 text-right text-status-success shrink-0`

### Visualizer Tab Layout

Single scrollable page, three sections stacked vertically with `xl` (32px) gap between sections:

```
+--------------------------------------------------+
| PROGRESS TREE                                    |
|  ▼ M001 — {milestone title}  [active, expanded] |
|     ▶ S01 — {slice title}    [active]            |
|        ○ T01 — {task title}  [pending]           |
|        ✔ T02 — {task title}  [done]              |
|  ▶ M002 — {milestone title}  [pending, collapsed]|
+--------------------------------------------------+
| COST & TOKENS                                    |
|  By Milestone                 By Model           |
|  M001 ■■■■■■■ $0.12          opus  ■■■■ $0.11   |
|  M002 ■■■ $0.04              sonnet ■■ $0.05     |
+--------------------------------------------------+
| EXECUTION TIMELINE                               |
|  {relative time}  S01 — {title}  $X.XX          |
|  {relative time}  T03 — {title}  $X.XX          |
+--------------------------------------------------+
```

Progress Tree:
- Milestone row: `flex items-center gap-2 py-2 cursor-pointer hover:bg-muted/30 rounded`
- Milestone chevron: `h-4 w-4 text-muted-foreground` — `ChevronRight` (collapsed) / `ChevronDown` (expanded)
- Milestone status icon: `StatusIcon` component (✔ / ▶ / ○ with color per status)
- Milestone label: `text-sm font-semibold`
- Slice row: `ml-6` indent; `text-sm`
- Task row: `ml-12` indent; `text-xs text-muted-foreground`

Cost bars:
- Two column layout: `grid grid-cols-2 gap-6` or stacked on narrow viewports
- Section label: `text-xs font-semibold text-muted-foreground uppercase tracking-wide mb-2`
- Bar row: `flex items-center gap-2 text-xs`
  - Key label: `w-24 truncate text-right text-muted-foreground`
  - Bar track: `flex-1 bg-muted rounded h-2 overflow-hidden`
  - Bar fill: `h-full bg-primary rounded` with `style={{ width: \`${pct}%\` }}`
  - Cost label: `w-16 text-right`

Timeline:
- Each entry: `flex items-center gap-3 py-2 border-b border-border/50 last:border-0 text-xs`
- Time: `w-24 text-muted-foreground shrink-0` (relative time via `formatRelativeTime()`)
- ID + title: `flex-1 truncate`
- Cost: `text-right text-muted-foreground shrink-0`

## States and Interactions

### Headless Tab States

| State | Visual |
|-------|--------|
| Loading (initial) | 3x `Skeleton` rows in snapshot area + log area |
| Idle — no prior session | Snapshot card shows "—" for all fields; log area shows empty state copy; Start button enabled, Stop button disabled |
| Idle — prior session completed | Snapshot card shows last snapshot; log shows prior run lines; "Last run {relative time}" inline |
| Running | Status dot green + `animate-pulse`; Stop button enabled (destructive); Start button disabled; log auto-scrolls |
| Complete | Status dot green solid; Stop button disabled; Start button enabled |
| Failed | Status dot red; error message in snapshot card state field; Start button enabled |
| Error loading | Error state card (see Copywriting Contract) |

### Start/Stop Button States

| Button | Idle | Running | Complete | Failed |
|--------|------|---------|---------|--------|
| Start Session | `variant="default"` enabled | `variant="default"` disabled | `variant="default"` enabled | `variant="default"` enabled |
| Stop Session | `variant="destructive"` disabled | `variant="destructive"` enabled | `variant="destructive"` disabled | `variant="destructive"` disabled |

### Visualizer Tab States

| State | Visual |
|-------|--------|
| Loading | `Skeleton` rows in each of the three sections |
| Empty (no metrics) | Empty state card in Cost & Tokens section (see Copywriting); Timeline section hidden |
| Populated | Full three-section layout; active milestone auto-expanded |
| Error loading | Error state card (see Copywriting) |

### Collapsible Tree Interaction

- Clicking milestone row header toggles expanded state
- Active milestone starts expanded; all others start collapsed
- Multiple milestones can be expanded simultaneously
- No animation on expand/collapse (keep render simple — CSS height transition not required)

### No Destructive Actions in This Phase

Stop Session terminates a process but does not delete data. No confirmation dialog is required. The Stop button uses `variant="destructive"` styling as a visual caution signal, not because it is irreversible.

## Pulsing Animation Contract

The active session status dot and active tree node icon both use `animate-pulse` (Tailwind built-in):

```
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: .5; }
}
animation: pulse 2s cubic-bezier(0.4, 0, 0.6, 1) infinite;
```

Usage:
- Running dot: `<span className="h-2 w-2 rounded-full bg-status-success animate-pulse" />`
- Active tree node: `<span className="text-yellow-500 animate-pulse">▶</span>`
- No custom keyframe needed — `animate-pulse` is already in Tailwind.
