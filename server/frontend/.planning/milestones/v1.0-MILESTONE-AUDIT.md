## Tech Debt Summary

### Total: 4 items across 3 phases

**Phase 06 — ROADMAP.md Checkbox (Trivial)**
- `.planning/ROADMAP.md` line 127: `- [ ] 06-01-PLAN.md` → `- [x] 06-01-PLAN.md`
- One-line fix. Not functional.

**Phase 06 — Missing Reactive Invalidation for Milestones/Slices/Tasks (~10 lines)**
- `use-gsd-file-watcher.ts` lines 104–111 must add invalidation of `gsd2Milestones`, `gsd2DerivedState`, `gsd2Milestone` (prefix), `gsd2Slice` (prefix)
- Without this: Milestones/Slices/Tasks tabs wait up to 30 seconds after `.gsd/` file changes
- Fix candidate for v2 or patch release

**Phase 05/01 — gsd2_get_roadmap_progress Unused in UI (Enhancement)**
- Rust command registered; TS wrapper exists (`tauri.ts`)
- No `useGsd2RoadmapProgress` hook, no `gsd2RoadmapProgress` query key, no UI component
- Data partially covered by health widget M/S/T counts
- Candidate for v2 progress bar feature

**Phase 01 — No Frontend Re-Detection Trigger (Edge Case)**
- `gsd2_detect_version` Tauri command exists but frontend never calls it
- Version is correctly set at import; only broken if project structure changes post-import
- Edge case; low impact
