## Overview

v1.0 establishes the structural foundation for GSD Cloud by integrating four existing projects (daemon, protocol-go, deployable-saas-template, gsd-vibe) into a unified monorepo. All three build pipelines (Go, pnpm, Docker Compose) pass from the repo root. No feature code was added — this milestone is purely structural.

## Milestone Summary

### Decimal Phases

None — no urgent insertions were required in v1.0.

### Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Monorepo with `server/` and `node/` directories | Separates deployment targets clearly: server = Docker Compose, node = Go binary | Validated — structure works cleanly |
| WebSocket JSON relay (not gRPC) | Protocol already defined and implemented in protocol-go | Maintained — no regressions |
| Comment out `frontend` Docker service | gsd-vibe was a Tauri desktop app with no Docker build path | Deferred to Phase 6 with TODO note |
| Add `useProjectWorkflows` stub | Missing hook blocked tsc; safe to stub since Tauri not present in web context | Validated — build unblocked, returns `undefined` gracefully |
| `go.work.sum` omission accepted | `go work sync` does not generate go.work.sum when module go.sum files cover all deps | Validated — correct Go toolchain behavior |

### Issues Resolved

- `deployable-saas-template/` residual untracked files after `git rm` — removed with `rm -rf`
- Frontend package named `vcca` (old internal name) — renamed to `server-frontend`
- `@tauri-apps/cli` devDep caused pnpm install failures on machines without Rust toolchain — removed
- `getProjectWorkflows` was a live Tauri IPC call (not a stub) — replaced with safe Promise.resolve() stub
- `.gitignore` missing `.env.production` and `.env.*` glob patterns — broadened to cover all variants
- `@tauri-apps/*` packages in `dependencies` instead of `devDependencies` — moved; prevents prod bundle bloat
- Dead `"tauri": "tauri"` script in `server/frontend/package.json` — removed

### Issues Deferred

| Issue | Deferred To | Reason |
|-------|-------------|--------|
| Frontend Docker service (Nginx + Vite build) | Phase 6 | No Dockerfile exists; gsd-vibe was Tauri-only |
| `useProjectWorkflows` REST API replacement | Phase 4 | Full Tauri-to-REST migration is Phase 4 scope |
| `packages/protocol-ts/` directory not pre-included in pnpm-workspace.yaml | Phase 3 | Path not needed until TypeScript protocol bindings are created |
| Adminer service undocumented (dev-only) | Phase 6 | Low priority; dev tooling concern |
| FastAPI async WebSocket backpressure | Phase 3 | Needs research before relay implementation |
| Tauri invoke() call audit for frontend migration scope | Phase 4 | Required before Phase 4 planning |
