## Overview

v1.0 delivers a self-hosted relay platform for managing remote Claude Code sessions from a mobile-first web UI. Four partially-built projects (daemon, protocol-go, deployable-saas-template, gsd-vibe) were integrated into a monorepo and extended with a full WebSocket relay chain: browser → FastAPI relay hub → Go daemon → Claude Code process. The milestone covers structural foundation, daemon reliability, server relay + auth, frontend integration, session persistence, and deployment polish — plus four gap-closure phases discovered during verification.

## Milestone Summary

**Key Decisions:**

- **Monorepo `server/` + `node/` split** — maps cleanly to Docker Compose vs Go binary deployment targets (✓ Validated)
- **WebSocket JSON relay, not gRPC** — protocol already implemented in protocol-go; no wire format change needed (✓ Validated)
- **In-memory ConnectionManager with Redis pub/sub fallback** — single-worker sufficient for v1; Redis added in Phase 6 for multi-worker (✓ Validated)
- **httpOnly cookie auth for browser WebSocket** — cannot use headers in browser WebSocket upgrade; JWT in cookie works cleanly (✓ Validated)
- **TDD xfail wave-0 stubs for Phase 3** — enabled test-first discipline across 32 test stubs before implementation (✓ Validated)
- **Asyncio Future pattern for node relay** — one-shot future per request ID for /fs and /file proxied responses with 5s timeout (✓ Validated)
- **Conditional cookie secure flag** — `secure=True` only when `ENVIRONMENT != "local"`; allows local HTTP dev without HTTPS (✓ Validated)

**Issues Resolved:**

- WAL race condition (RWMutex protecting PruneUpTo read-rewrite-rename atomically)
- Claude child process orphaning on daemon exit (pdeathsig on Linux)
- GET /nodes/{node_id} route missing (added in Phase 7 with ownership check)
- Browser WebSocket blocked in local dev by secure cookie requirement (fixed Phase 8)
- ReconnectionBanner unconnected to terminal component (wired Phase 9)
- VERIFICATION.md missing for phases 4 and 5 (created Phase 10)
- pytest running against SQLite instead of PostgreSQL (confirmed postgresql+psycopg in Phase 10)

**Technical Debt Incurred:**

- VIBE-02 gap: GSD Vibe project/phase/task/plan/roadmap screens are adapted UI shells; actual data flows through FastAPI but feature completeness wasn't verified end-to-end
- Milestone audit stale at Phase 7+ completion (audit ran before gap closure phases)
- No push notification support (deferred to v2 — NOTF-01/02/03)
- Redis pub/sub is wired but untested under multi-worker load
