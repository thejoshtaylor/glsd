## Critical Integration Gaps

### CRIT-01: No `channel_id` in `SessionPublic` — multi-session channelId collision

- **Affected:** SESS-01, SESS-03, SESS-06, RELY-02
- **Files:** `server/backend/app/models.py` (`SessionPublic`), `server/frontend/src/hooks/use-cloud-session.ts` lines 185–187
- **Problem:** Browser falls back to `channelId = sessionId`. Two sessions from the same browser tab collide on channelId.
- **Fix:** Add `channel_id: str` to `SessionPublic`; generate a UUID in `sessions.py` create endpoint.

### CRIT-02: `sendTask` sends `cwd: ''` — daemon starts Claude in undefined working directory

- **Affected:** SESS-01, SESS-03
- **File:** `server/frontend/src/hooks/use-cloud-session.ts` line 326
- **Problem:** `createSession` stores `cwd` in DB but `sendTask` hardcodes `cwd: ''`; `cwd` is not stored in session state.
- **Fix:** Store `cwd` in local state on `createSession` response; thread it into `sendTask`.

### CRIT-03: `GET /api/v1/nodes/{node_id}` route missing

- **Affected:** AUTH-05, AUTH-06, VIBE-04
- **Files:** `server/backend/app/api/routes/nodes.py` (missing), `server/frontend/src/lib/api/nodes.ts` line 34
- **Problem:** `useNode(nodeId)` calls `GET /nodes/:nodeId` → 404; NodeDetailPage renders error state.
- **Fix:** Add `@router.get("/{node_id}", response_model=NodePublic)` using `crud.get_node_by_id`.

### CRIT-04: WebSocket auth fails in local HTTP dev (cookie `secure=True` + no token query param)

- **Affected:** RELY-02, SESS-03, AUTH-02
- **Files:** `server/backend/app/api/routes/login.py` line 67, `server/frontend/src/lib/api/ws.ts` line 22
- **Problem:** `secure=True` cookie not sent over HTTP; `GsdWebSocket` doesn't append `token=` to WS URL; `ws_browser.py` closes with 1008.
- **Fix:** Make `secure=` conditional on env (`settings.ENVIRONMENT != "local"`) OR include JWT as `token=` query param in `GsdWebSocket.connect()`.

### CRIT-05: `listSessions(nodeId)` query param silently ignored by backend

- **Affected:** SESS-06, VIBE-04
- **Files:** `server/backend/app/api/routes/sessions.py` lines 37–43, `server/frontend/src/lib/api/sessions.ts` line 34
- **Problem:** Backend `list_sessions` accepts no `node_id` query param; always returns all user sessions.
- **Fix:** Add `node_id: UUID | None = None` query param to `list_sessions`; filter when provided.
