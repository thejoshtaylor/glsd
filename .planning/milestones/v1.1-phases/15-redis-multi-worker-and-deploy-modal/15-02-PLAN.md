    **initialNodeCount snapshot timing (critical -- per D-09):**
    Capture `initialNodeCount` inside the `useGeneratePairingCode` mutation's `onSuccess` callback. This is the exact moment to snapshot the count because: (a) the code has been confirmed generated server-side, and (b) the current `useNodes` data reflects the state BEFORE any new node could have paired. Implementation:
    ```typescript
    const nodesQuery = useQuery({
      queryKey: ['nodes'],
      queryFn: nodesApi.listNodes,
      refetchInterval: isPolling ? 3000 : false,
    });

    const generateCode = useGeneratePairingCode();

    const handleGenerate = async () => {
      const result = await generateCode.mutateAsync(nodeName);
      setCode(result.code);
      // Snapshot node count NOW -- before any new node could pair
      setInitialNodeCount(nodesQuery.data?.count ?? 0);
      setIsPolling(true);
      setStep(2);
    };
    ```

    Polling detection (per D-09):
    - On each refetch, compare `nodesQuery.data?.count > initialNodeCount`
    - If new node detected: set `isPolling=false`, call `onOpenChange(false)`, show toast "Node '<name>' connected." (per D-10)
    - Use a `useEffect` with a 10-minute timeout. On expiry: set `isExpired=true`, `isPolling=false`. Show "Expired -- regenerate" state with a button that resets to step 1 (per D-10).

    Note: The polling query shares queryKey `['nodes']` with `useNodes`, so React Query deduplicates. When `isPolling` is false, refetchInterval stops but the parent page's `useNodes` still works normally.

    On modal close (via X or backdrop), set `isPolling=false` to stop polling (Pitfall 6).

    Reset all state when modal opens (in onOpenChange handler or useEffect on open prop).
