# Phase 11.1 Plan 01: Wire Projects to Cloud API Summary

Projects page and dashboard wired to real GET /api/v1/projects endpoint via projectsApi.listProjects; both ProjectCard variants render slim ProjectPublic model (name, cwd, node_id, created_at) with path alias for backward compat.

## What Was Done

### Task 1: Wire queries.ts and adapt ProjectCard components

**queries.ts:**
- `useProjectsWithStats` and `useProjects` now call `projectsApi.listProjects()` and return `ProjectPublic[]`
- `useProject` calls `projectsApi.getProject(id)` instead of tauri stub

**api/projects.ts:**
- Added `path` field as alias for `cwd` via `enrichProject()` helper to maintain backward compat with components using `project.path`

**projects/project-card.tsx:**
- Complete rewrite: accepts `ProjectPublic`, renders name, cwd (truncated), node_id badge, created_at
- Removed all rich-stat rendering (git info, dependency status, favorites, todos, progress bars, cost)

**dashboard/project-card.tsx:**
- Same slim approach: name, node badge, cwd, created_at in fixed-height card
- Removed git info, favorites, tech stack badges, progress bars

**dashboard/project-row.tsx:**
- Slim list row: name, node badge, cwd, created_at

**projects.tsx:**
- Simplified to search-only filter (removed status/type filter selects)
- Removed bulk actions (BulkProjectBar, selectedIds, archive/delete handlers)
- Removed unused imports (useUpdateProject, useDeleteProject, design-tokens)

**dashboard.tsx:**
- Removed gitInfo batch queries, gitMap, status filter tabs
- Sort by created_at instead of is_favorite/last_activity_at
- Filter by name + cwd instead of name + description

**projects.test.tsx:**
- Updated mock data to use ProjectPublic shape
- Updated search placeholder to match new text

### Cascading Fixes (Rule 3 -- blocking tsc errors)

Changed return type of useProjectsWithStats from ProjectWithStats[] to ProjectPublic[] caused 40+ type errors in downstream consumers. Fixed all to compile clean:

- **status-bar.tsx**: Simplified to project count only (no tech_stack/gsd_version/path)
- **command-palette.tsx, breadcrumbs.tsx, main-layout.tsx**: Hardcoded hasPlanning/isGsd2 to false
- **main-layout.tsx**: recentProjects sorted by created_at instead of last_activity_at
- **knowledge-tab.tsx, unified-landing-view.tsx**: project passed as `any` where rich Project type expected
- **inbox.tsx**: Replaced all ProjectWithStats references with ProjectPublic, removed status/gsd_version filters
- **portfolio.tsx**: Same treatment -- status/cost/gsd_version references adapted
- **review.tsx**: Removed status filter
- **project.tsx**: tech_stack/gsd_version hardcoded false; ProjectHeader/ProjectOverviewTab get `as any` cast
