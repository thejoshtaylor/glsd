    ```python
    @router.post("/verify-email")
    def verify_email(session: SessionDep, token: str) -> Message:
        email = verify_email_verification_token(token=token)
        if not email:
            raise HTTPException(status_code=400, detail="Invalid or expired verification link")
        user = crud.get_user_by_email(session=session, email=email)
        if not user:
            raise HTTPException(status_code=400, detail="Invalid or expired verification link")
        if user.email_verified:
            return Message(message="Email already verified")
        user.email_verified = True
        user.email_verification_token = None
        session.add(user)
        session.commit()
        return Message(message="Email verified successfully")
    ```

    ```python
    @router.post("/resend-verification")
    def resend_verification(session: SessionDep, current_user: CurrentUser) -> Message:
        if current_user.email_verified:
            return Message(message="If your email needs verification, we sent a new link")
        # Rate limit: 60 second cooldown
        if current_user.email_verification_sent_at:
            elapsed = (datetime.now(timezone.utc) - current_user.email_verification_sent_at).total_seconds()
            if elapsed < 60:
                raise HTTPException(status_code=429, detail="Please wait before requesting another email")
        if settings.emails_enabled:
            token = generate_email_verification_token(email=current_user.email)
            current_user.email_verification_token = token
            current_user.email_verification_sent_at = datetime.now(timezone.utc)
            session.add(current_user)
            session.commit()
            email_data = generate_verification_email(email_to=current_user.email, token=token)
            send_email(email_to=current_user.email, subject=email_data.subject, html_content=email_data.html_content)
        return Message(message="If your email needs verification, we sent a new link")
    ```

    **2. Read-only enforcement dependency** (server/backend/app/api/deps.py):
    Add a new FastAPI dependency `require_verified_or_grace`:
    ```python
    from datetime import datetime, timedelta, timezone

    def require_verified_or_grace(current_user: CurrentUser) -> User:
        """Block write operations for unverified users past 7-day grace period.
        Inject on POST/PUT/PATCH/DELETE endpoints that modify user data."""
        if current_user.email_verified:
            return current_user
        if current_user.created_at is None:
            # Legacy user without created_at — allow
            return current_user
        grace_deadline = current_user.created_at + timedelta(days=7)
        if datetime.now(timezone.utc) > grace_deadline:
            raise HTTPException(
                status_code=403,
                detail="Email verification required. Please verify your email to restore full access."
            )
        return current_user

    VerifiedOrGraceDep = Annotated[User, Depends(require_verified_or_grace)]
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /verify-email | Untrusted token from URL query param |
| Browser -> /resend-verification | Authenticated but potentially unverified user |
| Browser -> /password-recovery/{email} | Unauthenticated email input |
| Browser -> /reset-password/ | Untrusted token + new password |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | /verify-email | mitigate | JWT with "purpose" claim prevents cross-use with reset tokens; 48h expiry; token stored on User model and cleared on use (single-use) |
| T-13-02 | Information Disclosure | /verify-email, /resend-verification | mitigate | Return generic "if your email needs verification" message regardless of user state; never reveal whether email exists |
| T-13-03 | Elevation of Privilege | Write endpoints | mitigate | require_verified_or_grace FastAPI dependency on POST/PUT/PATCH/DELETE endpoints; server-side enforcement, not frontend-only |
| T-13-04 | Denial of Service | /resend-verification | mitigate | 60-second cooldown via email_verification_sent_at timestamp check; 429 response |
| T-13-05 | Information Disclosure | Email send errors | mitigate | try/except wraps send_email in signup; logs warning server-side, never exposes SMTP details to client |
| T-13-06 | Spoofing | Token replay | mitigate | Token stored on User model; verification clears token; new token request overwrites old one |
</threat_model>
