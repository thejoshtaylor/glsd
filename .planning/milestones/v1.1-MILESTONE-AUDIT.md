- Accept as tech debt and proceed to `gsd-complete-milestone`

### 2. POST /nodes/code Missing VerifiedOrGraceDep (UX-01, AUTH-08)

**Severity:** HIGH — auth enforcement gap in node deployment flow

File: `server/backend/app/api/routes/nodes.py` line 66

`generate_pairing_code_endpoint` signature: `(body: NodeCodeRequest, current_user: CurrentUser)` — no `VerifiedOrGraceDep`.

`POST /api/v1/nodes/` (line 39) correctly uses `VerifiedOrGraceDep`. The pairing code endpoint was added in Phase 15 after Phase 13 applied the grace period dep, and it was not carried forward.

**Impact:** Unverified users past the 7-day grace period can generate pairing codes and deploy nodes — bypassing AUTH-08 grace period enforcement for this write operation.

**Fix:** One line — add `_verified: VerifiedOrGraceDep` to `generate_pairing_code_endpoint` signature (`VerifiedOrGraceDep` is already imported at line 17).
