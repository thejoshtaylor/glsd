Modify `get_current_user` in deps.py to accept a `Request` parameter and extract the JWT token using dual strategy:

1. First, try reading from `request.cookies.get("access_token")`
2. If no cookie, fall back to the Authorization header (`Authorization: Bearer <token>`)
3. If neither source provides a token, raise HTTP 401

Implementation details:
- Add `from fastapi import Request` to imports
- Change function signature to `def get_current_user(request: Request, session: SessionDep) -> User:`
- Remove the `token: TokenDep` parameter (no longer needed as a direct dependency)
- Keep `reusable_oauth2` and `TokenDep` defined — they are not harmful and may be used elsewhere
- Extract token with this logic:
  ```python
  token = request.cookies.get("access_token")
  if not token:
      auth_header = request.headers.get("authorization", "")
      if auth_header.startswith("Bearer "):
          token = auth_header[7:]
  if not token:
      raise HTTPException(status_code=401, detail="Not authenticated")
  ```
- Keep the existing jwt.decode + user lookup logic unchanged
- Change the jwt decode error status from 403 to 401 to match auth convention
- The `/me` endpoint in login.py can remain as-is (it has its own inline cookie logic and works fine)

This is the exact pattern used by the working `/me` endpoint in login.py, extended with Authorization header fallback so existing tests (which send `headers=superuser_token_headers`) continue to pass.
