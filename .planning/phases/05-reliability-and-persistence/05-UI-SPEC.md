## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` | Page background, terminal background |
| Secondary (30%) | `hsl(var(--card))` / `hsl(var(--muted))` | Activity feed widget panel, reconnection banner background |
| Accent (10%) | `hsl(var(--primary))` | Activity feed event count badge, "Reconnected" toast icon |
| Destructive | `hsl(var(--destructive))` | Not used in this phase (no destructive actions) |

Accent reserved for:
- Activity feed unread badge count (collapsed state)
- "Reconnected" toast success indicator
- Active session indicator dots in activity feed

Status colors used in activity feed events (not accent):
- `text-status-success` — taskComplete, session stopped (clean)
- `text-status-error` — taskError
- `text-status-warning` — permissionRequest (pending)
- `text-status-info` — question (pending)
- `text-gsd-cyan` — task sent, session created
- `text-muted-foreground` — session stopped (normal)

Source: existing `eventTypeColors` map in `activity-feed.tsx`, `globals.css` status color variables

## Component Inventory

### 1. Reconnection Banner (D-04)

**Location:** Above the terminal viewport, inside the session view.
**Dimensions:** Full width of terminal container, 32px height, 8px vertical padding.
**Background:** `hsl(var(--muted))` with `border-b border-border`.
**Content:** Centered row: 16px spinner (Lucide `Loader2` with `animate-spin`) + 8px gap + "Reconnecting..." at Label size (12px, weight 600 semibold, `text-muted-foreground`).
**Behavior:** Appears when `GsdWebSocket` state transitions to disconnected. Disappears when WS reconnects (before replay completes). Uses `animate-fade-in` on enter. No exit animation — instant removal.
**Terminal interaction:** Terminal remains visible below the banner. Input is disabled (pointer-events-none on input area, not on terminal output scroll). Banner does NOT push terminal content down — it overlays at position absolute top-0 within a relative container.

### 2. Reconnection Toast (D-05)

**Location:** Bottom-right corner (existing toast position in the app).
**Content:** "Reconnected" with a `CheckCircle2` icon (Lucide) in `text-status-success`, 14px body text.
**Duration:** 2000ms auto-dismiss.
**Trigger:** Fires after WS reconnection AND replay completion (all replayed events received). One toast per reconnection cycle — not per replayed message.
**Implementation:** Use existing toast infrastructure (project already uses toasts extensively — 22 files reference toast patterns).

### 3. Activity Feed Sidebar Widget (D-07, D-08, D-09, D-10)

**Location:** Right edge of main layout, collapsible panel.
**Collapsed state:** 40px-wide vertical strip with `Activity` icon (Lucide) centered, `aria-label="Open activity feed"`. When new events arrive while collapsed, show a small circular badge (16px diameter, `bg-primary text-primary-foreground`, 10px font) with unread count (cap display at "9+").
**Expanded state:** 320px-wide panel. Slides in from right with 150ms ease-out transition (`transform: translateX`). Contains:
- **Header:** 48px height. Left: "Activity" heading (14px semibold) + event count badge (`text-[10px] text-muted-foreground`). Right: collapse chevron button (`ChevronRight` icon, 32x32px touch target, `aria-label="Close activity feed"`).
- **Event list:** Scrollable area below header (`max-h: calc(100vh - 120px)`, `overflow-y: auto`). Maximum 100 events retained in memory (oldest dropped when exceeded). Each event item follows existing `ActivityItem` pattern from `activity-feed.tsx`:
  - Icon (14px, color-coded by event type) + 8px gap + message text (12px, truncated single line) + timestamp (10px, `text-muted-foreground`, relative time format).
  - Vertical divider between items: `divide-y divide-border/50`.
  - Clicking an event navigates to the relevant session view (if sessionId is present).
- **Empty state:** Centered text: "No activity yet" (12px, `text-muted-foreground`), with `Activity` icon above (32px, `text-muted-foreground/50`).

**Open/close trigger:** Icon button on the collapsed strip. Keyboard shortcut: none initially (discretion area — add later if requested). State persisted in React Context (not localStorage) — resets to collapsed on page refresh.

**Data source:** `EventSource` connecting to `GET /api/v1/activity/stream` (SSE). On mount, fetch initial batch via `GET /api/v1/activity?limit=50` with React Query. SSE prepends new events to the top of the list.

**Event type icon mapping** (extending existing `eventTypeIcons` from `activity-feed.tsx`):

| Event Type | Icon (Lucide) | Color Class |
|------------|---------------|-------------|
| task_sent | `Send` | `text-gsd-cyan` |
| taskComplete | `CheckCircle2` | `text-status-success` |
| taskError | `AlertTriangle` | `text-status-error` |
| permissionRequest | `ShieldQuestion` | `text-status-warning` |
| question | `MessageCircleQuestion` | `text-status-info` |
| session_created | `Play` | `text-gsd-cyan` |
| session_stopped | `Square` | `text-muted-foreground` |

**SSE error handling:** If SSE stream drops, reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s). No user-visible error — silent retry. After 3 consecutive failures, show a subtle inline message at the top of the event list: "Live updates paused — retrying" (12px, `text-status-warning`).

## Interaction States

### Reconnection Flow (SESS-05)

| State | Visual | Terminal | Input |
|-------|--------|----------|-------|
| Connected | No banner, no indicator | Live streaming | Enabled |
| Disconnected | Banner visible ("Reconnecting..." + spinner) | Frozen at last output | Disabled (pointer-events-none on prompt input) |
| Reconnecting (WS open, replay in progress) | Banner removed, replay events stream in | Output resumes (appended silently per D-02) | Disabled until replay completes |
| Reconnected | Toast appears (2s auto-dismiss) | Fully live | Enabled |

### Pending Control Messages on Reconnect (RELY-05, D-06)

When replayed events include a `permissionRequest` or `question` that has not been answered:
- The permission/question modal appears immediately, identical to live delivery.
- No special "replayed" indicator — the user sees the same modal they would see live.

### Activity Feed Widget (VIBE-06)

| State | Visual |
|-------|--------|
| Collapsed, no new events | 40px strip, Activity icon, no badge |
| Collapsed, new events | 40px strip, Activity icon, red badge with count |
| Expanded, loading | Skeleton shimmer (3 rows, matching existing `animate-shimmer` pattern) |
| Expanded, populated | Scrollable event list, newest at top |
| Expanded, empty | Centered empty state with icon + copy |
| SSE disconnected | Inline warning at top of event list |
