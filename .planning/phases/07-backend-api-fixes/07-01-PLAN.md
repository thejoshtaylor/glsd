<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> GET /nodes/{node_id} | Untrusted node_id path parameter from authenticated user |
| Browser -> GET /sessions/?node_id= | Untrusted node_id query parameter from authenticated user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | GET /nodes/{node_id} | mitigate | Ownership check via `crud.get_node_by_id(user_id=current_user.id)` -- returns None if node not owned by user. Route returns 404 for both "not found" and "not owned" (no information leak about existence of other users' nodes). Test: `test_get_other_users_node` asserts 404. |
| T-07-02 | Information Disclosure | GET /sessions/?node_id= | mitigate | Filter query includes `WHERE user_id = current_user.id` in all cases. Even if attacker guesses a valid node_id belonging to another user, the sessions query returns empty (not 403). Test: `test_list_sessions_by_node` creates sessions for one node and verifies isolation. |
| T-07-03 | Spoofing | node_id path/query parameter | mitigate | UUID parsing via `uuid.UUID(node_id)` in try/except. Invalid UUIDs return 404 (nodes) or empty list (sessions). No SQL injection risk -- SQLModel parameterizes queries. |
| T-07-04 | Tampering | channel_id on SessionPublic | accept | channel_id is read-only (set to None on REST responses). It is not accepted as input on any endpoint. No tamper vector. |
</threat_model>
