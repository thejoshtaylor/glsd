**Backend — cookie auth (per D-04):**

1. In `server/backend/app/api/routes/login.py`, add a new endpoint `POST /api/v1/login/cookie` that:
   - Accepts `OAuth2PasswordRequestForm` (same as existing login)
   - Authenticates via `crud.authenticate(session=session, email=form_data.username, password=form_data.password)`
   - On success, creates JWT via `security.create_access_token(user.id, expires_delta=timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES))`
   - Returns `JSONResponse(content={"email": user.email, "id": str(user.id)})` with `response.set_cookie(key="access_token", value=token, httponly=True, secure=True, samesite="lax", max_age=int(access_token_expires.total_seconds()), path="/")`
   - Also add `POST /api/v1/logout` that returns `JSONResponse` with `response.delete_cookie(key="access_token", path="/")`
   - Also add `GET /api/v1/me` that reads `access_token` cookie, decodes JWT, returns user info

2. In `server/backend/app/api/routes/ws_browser.py`, update the token extraction to read from cookie first, fallback to query param:
   ```python
   token = websocket.cookies.get("access_token", "")
   if not token:
       token = websocket.query_params.get("token", "")
   ```
   Keep rest of the auth logic identical.

**Frontend — API client foundation (per D-01):**

8. Create `server/frontend/src/lib/api/ws.ts` — the `GsdWebSocket` class:
   - Constructor takes no args
   - `connect(channelId: string): void` — opens `${location.protocol === 'https:' ? 'wss:' : 'ws:'}//${location.host}/ws/browser?channelId=${channelId}` (no token param — cookie auth per D-04)
   - `on(type: string, handler: (msg: ProtocolMessage) => void): () => void` — registers handler, returns unsubscribe function
   - `send(msg: object): void` — JSON.stringify and send
   - `disconnect(): void` — close WebSocket, clear handlers
   - Private `scheduleReconnect(channelId: string)` — exponential backoff: `Math.min(1000 * 2 ** this.reconnectAttempts, 30000)` delay
   - `onmessage` handler: `JSON.parse(event.data)` then dispatch to matching type handlers
   - `onopen`: reset reconnectAttempts to 0
   - `onclose`: call scheduleReconnect

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> FastAPI REST | All REST calls authenticated via httpOnly cookie |
| Browser -> FastAPI WS | WebSocket upgrade authenticated via cookie JWT |
| Frontend -> Backend | CORS boundary; credentials: include required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | login.py cookie endpoint | mitigate | Set httponly=True, secure=True, samesite=lax on access_token cookie |
| T-04-02 | Tampering | ws_browser.py cookie read | mitigate | JWT signature validation before accept(); cookie fallback to query param only during migration |
| T-04-03 | Info Disclosure | client.ts error handling | mitigate | Do not expose server stack traces to UI; only show body.detail string |
| T-04-04 | Elevation of Privilege | client.ts 401 handling | mitigate | Redirect to /login on 401; do not retry with stale credentials |
</threat_model>
