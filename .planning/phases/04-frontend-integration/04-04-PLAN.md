| NodesPage -> /api/v1/nodes | Node list filtered by authenticated user on server |
| NodeDetailPage -> /api/v1/nodes/:id/revoke | Server validates node ownership before revoke |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Info Disclosure | nodes-page.tsx | mitigate | Server only returns nodes belonging to authenticated user (Phase 3 ownership filter) |
| T-04-14 | Elevation of Privilege | node-detail-page.tsx revoke | mitigate | Server validates node ownership before processing revoke; frontend adds confirmation dialog as UX layer |
| T-04-15 | Tampering | node-detail-page.tsx URL nodeId | accept | Server rejects requests for nodes not owned by user; 404 response handled gracefully in UI |
</threat_model>
