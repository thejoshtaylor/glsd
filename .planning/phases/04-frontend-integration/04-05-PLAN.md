**Backend filesystem proxy endpoint (per D-13, Blocker 1B):**

1. Add to `server/backend/app/api/routes/nodes.py` a new endpoint `GET /nodes/{node_id}/fs`:
   - Query param: `path: str = "/"` (use FastAPI `Query` parameter)
   - Requires `CurrentUser` dependency (authenticated)
   - Validates node belongs to user: query DB for Node with matching id and user_id, raise HTTPException(404) if not found
   - Check node is not revoked: raise HTTPException(400, "Node is revoked") if `node.is_revoked`
   - Check node is connected: validate node has a `machine_id` and `manager.is_node_connected(node.machine_id)` (or check if machine_id is in manager's node connections), raise HTTPException(503, "Node is offline") if not connected
   - Sends a `browseDir` message to the node via WebSocket relay:
     ```python
     import asyncio
     import uuid as uuid_mod

     request_id = str(uuid_mod.uuid4())
     msg = {
         "type": "browseDir",
         "requestId": request_id,
         "channelId": request_id,  # Use requestId as channelId for one-shot request-response
         "machineId": str(node.machine_id),
         "path": path,
     }
     ```
   - Uses an `asyncio.Future` to await the response:
     - Register a one-shot response callback on the ConnectionManager keyed by `request_id`
     - Call `manager.send_to_node(node.machine_id, msg)` to send the browseDir to the node
     - `result = await asyncio.wait_for(future, timeout=5.0)` — 5 second timeout
     - On `browseDirResult` message from node matching `requestId`, resolve the future with the message payload
   - Returns JSON: `{ "entries": result["entries"] }` (pass through the entries array from browseDirResult)
   - If `result["ok"]` is False: return 400 with `{ "detail": result.get("error", "Browse failed") }`
   - On timeout (asyncio.TimeoutError): return `HTTPException(status_code=504, detail="Node did not respond")`
   - On node not connected: return `HTTPException(status_code=503, detail="Node is offline")`

   Note: The ConnectionManager may need a small helper method to register a one-shot response handler. If `manager` does not already have a `register_response_handler(request_id, future)` and `resolve_response(request_id, message)` pattern, add it:
   ```python
   # In ConnectionManager:
   _pending_responses: dict[str, asyncio.Future] = {}

   def register_response(self, request_id: str) -> asyncio.Future:
       loop = asyncio.get_event_loop()
       future = loop.create_future()
       self._pending_responses[request_id] = future
       return future

   def resolve_response(self, request_id: str, message: dict) -> None:
       future = self._pending_responses.pop(request_id, None)
       if future and not future.done():
           future.set_result(message)
   ```
   Then in the node message handler (where incoming node messages are processed), add a check:
   ```python
   if msg.get("type") == "browseDirResult":
       manager.resolve_response(msg.get("requestId", ""), msg)
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /api/v1/nodes/:id/fs | Path parameter from user; potential directory traversal |
| Server -> Node (browseDir) | Server relays path to node; node must validate |
| File browser UI | Renders filenames from untrusted node response |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-16 | Tampering | /nodes/:id/fs path param | mitigate | Server validates node ownership. Node daemon must sanitize path to prevent traversal above cwd (daemon responsibility from Phase 2). Server passes path through without transformation. |
| T-04-17 | Info Disclosure | file-browser.tsx | mitigate | File names rendered as text content (not innerHTML). File content displayed in pre block with no script execution. |
| T-04-18 | Denial of Service | /nodes/:id/fs timeout | mitigate | 5-second asyncio.wait_for timeout prevents hung requests; 504 returned on timeout |
| T-04-19 | Spoofing | browseDir channelId | mitigate | Server uses random UUID as channelId for filesystem requests; response matched by exact requestId |
</threat_model>
