**Backend filesystem proxy (per D-13, Research Pitfall 6):**

1. Add to `server/backend/app/api/routes/nodes.py` a new endpoint `GET /api/v1/nodes/{node_id}/fs`:
   - Query param: `path` (string, default="/")
   - Requires `CurrentUser` dependency (authenticated)
   - Validates node belongs to user (same ownership check as other node endpoints)
   - Validates node is online via `manager.is_node_connected(node_id)`
   - Sends a `browseDir` message to the node via WebSocket:
     ```python
     import asyncio
     import uuid

     request_id = str(uuid.uuid4())
     msg = {
         "type": "browseDir",
         "sessionId": "",
         "channelId": request_id,
         "path": path,
     }
     ```
   - Uses an `asyncio.Future` to await the response:
     - Register a response callback on the ConnectionManager keyed by `request_id`
     - `await asyncio.wait_for(future, timeout=10.0)` — 10 second timeout
     - On `browseDirResult` message from node matching `channelId == request_id`, resolve the future
   - Returns JSON: `{ entries: [{ name: string, is_dir: boolean, size: number, mod_time: string }] }`
   - On timeout: return 504 with `{ detail: "Node did not respond" }`
   - On node not connected: return 503 with `{ detail: "Node is offline" }`

   Note: The ConnectionManager may need a small helper method to register a one-shot response handler keyed by channelId/requestId. Add `register_response_handler(channel_id, callback)` and `handle_node_message_for_responses(msg)` to ConnectionManager if not already present.

**Frontend API function:**

2. Add to `server/frontend/src/lib/api/nodes.ts`:
   ```typescript
   export interface FsEntry {
     name: string;
     is_dir: boolean;
     size: number;
     mod_time: string;
   }

   export function browseNodeFs(nodeId: string, path: string = '/'): Promise<{ entries: FsEntry[] }> {
     return apiRequest<{ entries: FsEntry[] }>(`/nodes/${nodeId}/fs?path=${encodeURIComponent(path)}`);
   }
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /api/v1/nodes/:id/fs | Path parameter from user; potential directory traversal |
| Server -> Node (browseDir) | Server relays path to node; node must validate |
| File browser UI | Renders filenames from untrusted node response |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-16 | Tampering | /nodes/:id/fs path param | mitigate | Server validates node ownership. Node daemon must sanitize path to prevent traversal above cwd (daemon responsibility from Phase 2). Server passes path through without transformation. |
| T-04-17 | Info Disclosure | file-browser.tsx | mitigate | Only directory listings displayed; file contents not loaded in this phase. File names rendered as text content (not innerHTML). |
| T-04-18 | Denial of Service | /nodes/:id/fs timeout | mitigate | 10-second asyncio.wait_for timeout prevents hung requests; 504 returned on timeout |
| T-04-19 | Spoofing | browseDir channelId | mitigate | Server uses random UUID as channelId for filesystem requests; response matched by exact channelId |
</threat_model>
