**useCloudSession hook (per D-03):**

1. Create `server/frontend/src/hooks/use-cloud-session.ts`:

State type:
```typescript
interface CloudSessionState {
  sessionId: string | null;
  channelId: string | null;
  isConnected: boolean;
  isLoading: boolean;
  error: string | null;
  pendingPermission: PermissionRequestMessage | null;
  pendingQuestion: QuestionMessage | null;
}
```

Hook interface:
```typescript
interface UseCloudSessionOptions {
  onData?: (data: string) => void;        // stream text -> write to xterm
  onTaskComplete?: (result: any) => void;
  onTaskError?: (error: string) => void;
  onError?: (error: string) => void;
}

interface UseCloudSessionReturn {
  state: CloudSessionState;
  createSession: (nodeId: string, cwd: string) => Promise<string>;
  sendTask: (prompt: string, opts?: { model?: string; effort?: string; permissionMode?: string }) => void;
  sendStop: () => void;
  respondPermission: (requestId: string, approved: boolean) => void;
  respondQuestion: (requestId: string, answer: string) => void;
  disconnect: () => void;
}
```

Implementation:
- `createSession(nodeId, cwd)`:
  - Call `sessionsApi.createSession(nodeId, cwd)` to get `{ id: sessionId, channel_id: channelId }`
  - Create new `GsdWebSocket` instance, call `ws.connect(channelId)`
  - Register handlers:
    - `ws.on('stream', (msg) => { onData?.(extractStreamText(msg.event)); })` — extract text content from Claude event objects. At minimum handle: if `event.type === 'content_block_delta'` and `event.delta?.text`, pass that text. If `event.type === 'text'`, pass `event.text`. Fallback: `JSON.stringify(event)` for unrecognized event shapes.
    - `ws.on('taskStarted', () => { set isConnected true })` 
    - `ws.on('taskComplete', (msg) => { onTaskComplete?.(msg); set state accordingly })`
    - `ws.on('taskError', (msg) => { onTaskError?.(msg.error); set error })`
    - `ws.on('permissionRequest', (msg) => { set pendingPermission to msg })`
    - `ws.on('question', (msg) => { set pendingQuestion to msg })`
  - Store ws ref and sessionId/channelId in state
  - Return sessionId

- `sendTask(prompt, opts)`:
  - Build TaskMessage: `{ type: 'task', taskId: crypto.randomUUID(), sessionId: state.sessionId, channelId: state.channelId, prompt, model: opts?.model || 'claude-sonnet-4-20250514', effort: opts?.effort || 'high', permissionMode: opts?.permissionMode || 'default', cwd: '' }`
  - Call `ws.send(taskMessage)`

- `sendStop()`:
  - Build StopMessage: `{ type: 'stop', sessionId: state.sessionId, channelId: state.channelId }`
  - Call `ws.send(stopMessage)`

- `respondPermission(requestId, approved)`:
  - Build PermissionResponseMessage: `{ type: 'permissionResponse', sessionId: state.sessionId, channelId: state.channelId, requestId, approved }`
  - Call `ws.send(msg)`
  - Set `pendingPermission` to null

- `respondQuestion(requestId, answer)`:
  - Build QuestionResponseMessage: `{ type: 'questionResponse', sessionId: state.sessionId, channelId: state.channelId, requestId, answer }`
  - Call `ws.send(msg)`
  - Set `pendingQuestion` to null

- `disconnect()`: call `ws.disconnect()`, reset state

- Cleanup: on unmount, call disconnect

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| useCloudSession -> WebSocket | All messages authenticated via cookie on WS upgrade |
| PermissionPrompt -> user decision | User explicitly approves/denies; no auto-approve |
| Stream event parsing | Untrusted event data from Claude rendered in xterm |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering | use-cloud-session.ts sendTask | mitigate | Server validates session ownership; channelId overwritten server-side (per Phase 3 T-03-19) |
| T-04-10 | Info Disclosure | permission-prompt.tsx | mitigate | toolInput JSON displayed read-only; no editable fields that could be submitted back |
| T-04-11 | Spoofing | use-cloud-session.ts respondPermission | mitigate | requestId comes from server-validated permissionRequest; server matches requestId to session |
| T-04-12 | Denial of Service | stream event parsing | accept | Malformed events fall back to JSON.stringify display; no crash path |
</threat_model>
