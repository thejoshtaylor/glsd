**ProtectedRoute (per D-05):**

1. Create `server/frontend/src/components/auth/protected-route.tsx`:
   ```typescript
   import { Navigate, useLocation } from 'react-router-dom';
   import { useAuth } from '@/contexts/auth-context';
   import { Loader2 } from 'lucide-react';

   export function ProtectedRoute({ children }: { children: React.ReactNode }) {
     const { user, isLoading } = useAuth();
     const location = useLocation();

     if (isLoading) {
       return (
         <div className="flex items-center justify-center h-screen">
           <Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />
         </div>
       );
     }

     if (!user) {
       return <Navigate to="/login" state={{ from: location }} replace />;
     }

     return <>{children}</>;
   }
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser localStorage | No tokens stored; auth is cookie-only |
| /login form -> /api/v1/login/cookie | Credentials submitted over HTTPS |
| ProtectedRoute -> AuthContext | Client-side gate; server enforces auth independently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Spoofing | login-page.tsx | mitigate | Form submits to httpOnly cookie endpoint; no token in JavaScript memory or localStorage |
| T-04-06 | Info Disclosure | auth-context.tsx | mitigate | Only store user email+id in context; no token or password in React state |
| T-04-07 | Denial of Service | login-page.tsx | accept | Rate limiting is a server concern (Phase 3 backend); frontend shows error on 429 |
| T-04-08 | Elevation of Privilege | protected-route.tsx | mitigate | Client-side ProtectedRoute is UX only; server validates cookie JWT on every API call independently |
</threat_model>
