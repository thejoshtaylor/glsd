**server/frontend/nginx.conf** -- Create this file:
```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # API proxy -- forward to FastAPI backend
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket proxy -- forward to FastAPI backend
    location /ws/ {
        proxy_pass http://backend:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }

    # SPA fallback -- serve index.html for all non-file routes
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
}
```

**server/frontend/Dockerfile** -- Create a multi-stage Dockerfile. The build context will be set to the repo root (`..` from `server/`) so we can access `pnpm-lock.yaml` and `pnpm-workspace.yaml`:

```dockerfile
# Stage 1: Build the React/Vite app
FROM node:22-alpine AS build

RUN corepack enable

WORKDIR /app

# Copy workspace config and lock file from repo root
COPY pnpm-workspace.yaml pnpm-lock.yaml ./

# Copy frontend package.json
COPY server/frontend/package.json ./server/frontend/

# Install dependencies
RUN cd server/frontend && pnpm install --frozen-lockfile

# Copy frontend source
COPY server/frontend/ ./server/frontend/

# Build the app (output to server/frontend/dist/)
RUN cd server/frontend && pnpm build

# Stage 2: Serve with Nginx
FROM nginx:alpine

# Remove default nginx config
RUN rm /etc/nginx/conf.d/default.conf

# Copy custom nginx config
COPY server/frontend/nginx.conf /etc/nginx/conf.d/default.conf

# Copy built static files
COPY --from=build /app/server/frontend/dist /usr/share/nginx/html

EXPOSE 80
```

**Key decisions per D-10:** No `VITE_API_URL` build arg needed -- frontend uses relative `/api/v1` paths which Nginx proxies to the backend. The `EXPOSE 80` is documentation only (no ports published in docker-compose.yml).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Nginx | Untrusted HTTP requests enter through Nginx |
| Nginx -> Backend | Proxied requests to internal backend service |
| Nginx -> Filesystem | Static file serving from built frontend assets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Spoofing | Nginx header injection | mitigate | Nginx `proxy_set_header` explicitly sets Host, X-Real-IP, X-Forwarded-For, X-Forwarded-Proto -- overriding any client-sent values. Backend trusts only these headers. |
| T-06-06 | Information Disclosure | Nginx path traversal | mitigate | `try_files $uri $uri/ /index.html` only serves files within `/usr/share/nginx/html` (the build output). Nginx's `root` directive constrains file access. No `alias` directive used (which is more prone to traversal). |
| T-06-07 | Spoofing | SSRF via proxied paths | mitigate | `proxy_pass` targets are hardcoded to `http://backend:8000` (Docker DNS). No user-controlled proxy targets. Only `/api/` and `/ws/` prefixes are proxied; all other paths serve static files. |
| T-06-08 | Information Disclosure | Docker build secrets in image layers | mitigate | No secrets in Dockerfile. No .env files copied into image. Build uses only package.json, source code, and lock files. Multi-stage build means Node toolchain is not in final image. |
| T-06-09 | Tampering | Clickjacking via iframe embedding | mitigate | Nginx adds `X-Frame-Options: SAMEORIGIN` header. Also adds `X-Content-Type-Options: nosniff` and `Referrer-Policy`. |
</threat_model>
