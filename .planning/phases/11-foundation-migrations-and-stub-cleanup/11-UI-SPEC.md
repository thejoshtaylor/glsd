# Phase 11 -- UI Design Contract

> Visual and interaction contract for Phase 11: Foundation -- Migrations and Stub Cleanup.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Character

Phase 11 is a **backend/infrastructure stabilization** phase. It introduces **no new screens, no new visual components, and no new layouts**. All UI work consists of:

1. Making existing pages (Nodes, Projects) stop 500-erroring after migration
2. Replacing Tauri stub calls with real server API calls (existing UI, new wiring)
3. Surfacing email send errors as toast notifications

The design contract below locks the error-state copywriting and confirms the existing design system tokens that implementors must use. No new design tokens or components are introduced.

## Copywriting Contract

Phase 11 has no primary CTA or empty states. The copywriting contract covers error states surfaced by FIX-03 (email hardening) and Tauri stub replacements.

### Email Error Toasts (FIX-03)

| Element | Copy |
|---------|------|
| SMTP not configured (503) | Title: "Email not available" / Body: "Email sending is not configured on this server. Contact your server admin." |
| Email send failed (502) | Title: "Email failed to send" / Body: "The server could not deliver the email. Check SMTP settings and try again." |

### Tauri Stub Replacement Errors (FIX-02)

These are error states that may surface when a Tauri stub is replaced with a real server API call and that call fails.

| Element | Copy |
|---------|------|
| Project path check failed | Title: "Could not verify path" / Body: "The server could not access the specified project path. Verify the path exists on the node." |
| Project scaffold failed | Title: "Project creation failed" / Body: "Could not scaffold the project. Check the server logs for details." |
| File read failed | Title: "Could not load file" / Body: "The server could not read the requested file. The node may be disconnected." |
| Generic API error (fallback) | Title: "Something went wrong" / Body: "An unexpected error occurred. Try again or check the server logs." |

### Toast Pattern

All error toasts use `sonner` (established pattern). Implementation:

```typescript
// Email 503
toast.error("Email not available", {
  description: "Email sending is not configured on this server. Contact your server admin.",
});

// Email 502
toast.error("Email failed to send", {
  description: "The server could not deliver the email. Check SMTP settings and try again.",
});
```

### Destructive Actions

Phase 11 introduces no new destructive actions. Existing destructive patterns (node deletion, project removal) are unchanged.
