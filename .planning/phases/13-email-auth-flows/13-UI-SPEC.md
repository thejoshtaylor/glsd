## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, form field gaps, label-to-input spacing |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card header/content padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none. The existing login-page.tsx uses `space-y-1.5` (6px) for label-to-input spacing — this is a legacy value and must NOT be carried forward to new auth pages. New auth pages must use `space-y-2` (8px) for label-to-input spacing to stay on the 4px grid. Login page uses `space-y-4` (16px) for form field groups — follow this pattern for all new auth pages.

Source: login-page.tsx uses `space-y-4`, `gap-1`, `pb-4`, `mt-3`, `mb-2` [VERIFIED: codebase]

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Heading | 18px (`text-lg`) | 600 (semibold) | 1.4 |
| Subtext | 13px (custom `font-scale-md`) | 400 (regular) | 1.5 |

Notes:
- Only two font weights are used: 400 (regular) for body, labels, and subtext; 600 (semibold) for headings only.
- Labels are distinguished from body text by their semantic role and proximity to inputs, not by weight. Use the `Label` component from `@/components/ui/label` which provides the correct styling.
- Login page uses `text-lg` for CardTitle — follow this pattern for all auth page headings.
- Muted helper text uses `text-sm text-muted-foreground`.
- Banner action link uses `text-sm text-primary` (differentiated by color, not weight).

Source: login-page.tsx, globals.css font-scale presets [VERIFIED: codebase]

|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` | Page background (`bg-background`) |

## Page Specifications

### ForgotPasswordPage (`/forgot-password`)

**Layout:** Centered card, identical to login page layout (`min-h-screen flex items-center justify-center bg-background p-4`).

**States:**

1. **Default (form):**
   - GSD logo + "GSD Cloud" heading (same as login page)
   - Subheading: "Reset your password"
   - Email input field with label "Email"
   - Submit button: "Send Reset Link"
   - Back link: "Back to Sign In" (text link, `text-sm text-muted-foreground hover:text-primary`, navigates to `/login`)

2. **Submitting:**
   - Button shows Loader2 spinner + "Sending..."
   - Input and button disabled

3. **Success (submitted):**
   - Replace form with success message
   - Icon: `Mail` from lucide-react, `h-12 w-12 text-primary`, centered
   - Heading: "Check your email"
   - Body: "If an account exists for that email, we sent a password reset link. Check your inbox and spam folder."
   - Link: "Back to Sign In" (same styling as above)

4. **Error:**
   - Toast via `sonner`: `toast.error("Something went wrong. Please try again.")`
   - Form remains interactive for retry

### ResetPasswordPage (`/reset-password`)

**Layout:** Same centered card as ForgotPasswordPage.

**Token extraction:** Read `token` from URL query parameter (`?token=xxx`).

**States:**

1. **Default (form):**
   - GSD logo + "GSD Cloud" heading
   - Subheading: "Set a new password"
   - Password input with label "New Password" (minLength 8)
   - Confirm password input with label "Confirm Password"
   - Submit button: "Reset Password"

2. **Submitting:**
   - Button shows Loader2 spinner + "Resetting..."
   - Inputs and button disabled

3. **Success:**
   - Replace form with success message
   - Icon: `CheckCircle` from lucide-react, `h-12 w-12 text-primary`, centered
   - Heading: "Password reset"
   - Body: "Your password has been updated. You can now sign in with your new password."
   - Link: "Sign In" (navigates to `/login`)

4. **Invalid/expired token:**
   - Icon: `XCircle` from lucide-react, `h-12 w-12 text-destructive`, centered
   - Heading: "Invalid or expired link"
   - Body: "This password reset link is no longer valid. Please request a new one."
   - Link: "Request New Link" (navigates to `/forgot-password`)

5. **Password mismatch (client-side):**
   - Inline error below confirm field: `text-sm text-destructive` — "Passwords do not match"
   - Submit button remains disabled until match

### VerifyEmailPage (`/verify-email`)

**Layout:** Same centered card.

**Token extraction:** Read `token` from URL query parameter.

**States:**

1. **Verifying (on mount):**
   - GSD logo + "GSD Cloud" heading
   - Loader2 spinner centered, `h-8 w-8 text-primary animate-spin`
   - Text: "Verifying your email..."

2. **Success:**
   - Icon: `CheckCircle` from lucide-react, `h-12 w-12 text-primary`, centered
   - Heading: "Email verified"
   - Body: "Your email has been verified. You now have full access to GSD Cloud."
   - Link: "Continue to Dashboard" (navigates to `/`)

3. **Already verified:**
   - Icon: `CheckCircle` from lucide-react, `h-12 w-12 text-muted-foreground`, centered
   - Heading: "Already verified"
   - Body: "This email has already been verified."
   - Link: "Go to Dashboard" (navigates to `/`)

4. **Invalid/expired token:**
   - Icon: `XCircle` from lucide-react, `h-12 w-12 text-destructive`, centered
   - Heading: "Invalid or expired link"
   - Body: "This verification link is no longer valid. You can request a new one from your dashboard."
   - Link: "Go to Dashboard" (navigates to `/`)

### EmailVerificationBanner

**Placement:** Rendered at top of main layout content area, inside ProtectedRoute, above page content. Only shown when `user.email_verified === false`.

**Visual:**
- Full-width bar with `bg-status-warning/10 border border-status-warning/20 rounded-md` inside the content area
- Horizontal layout: icon + text + action on one line
- Icon: `AlertTriangle` from lucide-react, `h-4 w-4 text-status-warning`
- Text (standard, days 0-6): "Please verify your email address to maintain full access."
- Text (urgent, day 7+): "Your account is in read-only mode. Verify your email to restore full access."
- Action: "Resend Verification Email" button — `variant="link"` style, `text-primary text-sm`
- Dismiss: no dismiss button — banner persists until verified

**Resend behavior:**
- On click: call `/resend-verification` API
- While sending: button text changes to "Sending..."
- On success: `toast.success("Verification email sent. Check your inbox.")`
- On rate limit (too soon): `toast.error("Please wait before requesting another email.")`
- Cooldown: disable button for 60 seconds after successful send

## Interaction Contracts

### Form Validation

All auth forms use the same validation pattern from login-page.tsx:

| Rule | Implementation |
|------|----------------|
| Required fields | HTML `required` attribute |
| Email format | `type="email"` native validation |
| Password minimum | `minLength={8}` on input |
| Password match | Client-side comparison, inline error text |
| Disable on submit | All inputs and button `disabled={isSubmitting}` |

### Navigation Flow

```
/login
  |-- "Forgot password?" link --> /forgot-password
  
/forgot-password
  |-- "Back to Sign In" link --> /login
  |-- Submit (success) --> success message with "Back to Sign In" link
  
/reset-password?token=xxx  (from email link)
  |-- Submit (success) --> success message with "Sign In" link --> /login
  |-- Invalid token --> error message with "Request New Link" --> /forgot-password
  
/verify-email?token=xxx  (from email link)
  |-- Auto-verify on mount --> success/error with dashboard link --> /

EmailVerificationBanner (in authenticated layout)
  |-- "Resend Verification Email" --> API call, toast feedback
```

### Login Page Addition

Add a "Forgot password?" link to the existing login page:
- Position: below the password field, right-aligned
- Styling: `text-sm text-muted-foreground hover:text-primary transition-colors`
- Copy: "Forgot password?"
- Target: `/forgot-password`

### AuthUser Interface Extension

Extend `AuthUser` in auth-context.tsx:

```typescript
export interface AuthUser {
  email: string;
  id: string;
  email_verified: boolean;  // NEW
  created_at: string;       // NEW — ISO date, for 7-day grace period calculation
}
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Focus management | Auto-focus email input on page load (`autoFocus` on first field) |
