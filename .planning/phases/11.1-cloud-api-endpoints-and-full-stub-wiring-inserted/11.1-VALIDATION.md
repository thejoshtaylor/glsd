## Validation Approach

This phase uses a combination of:

1. **TypeScript compilation checks** (`tsc --noEmit`) — catches type mismatches between frontend components and API response shapes (Plans 01, 03)
2. **Python import checks** — verifies models and routes load without syntax/import errors (Plans 02, 03)
3. **Integration tests** — `test_user_settings.py` covers D-03/D-05 settings CRUD; `test_project_filters.py` covers D-08/D-09/D-10 backend filter logic with behavioral assertions
4. **Full backend test suite** — run after each wave to catch regressions

Wave 0 stub test files are NOT used. Instead, each plan creates its own test file inline:
- Plan 02 creates `test_user_settings.py` (4 tests: defaults, idempotent, update, unauth)
- Plan 03 creates `test_project_filters.py` (5 tests: activity/sessions/usage filter + invalid UUID handling)
